Hard rules:

1. `motivation.why_prior_stopped` MUST cite ≥ 2 specific Phase 0 paper_ids with all four sub-fields populated. Generic "prior work didn't do X" without paper attributions is a process error.

2. `method_flow.steps` MUST be numbered (S1, S2, ...) with `linked_component` AND `linked_falsification` per step. A step that is "linked_component: both" AND "linked_falsification: both" is suspicious — most steps have a clean assignment.

3. `falsification_prediction.what_we_run`, `falsification_prediction.compute_budget`, and `mechanism_probe_proposal` are echoed VERBATIM from the Phase 2.2 candidate (or Phase 3.3 final_candidate when revise ran — both equal to Phase 2.2 by anti-substitution rule). Phase 4 cannot quietly substitute a friendlier procedure. The kill_switch_integrity validator rejects drift.

4. `abstract_draft` must contain (in order): problem → bottleneck (cite ≥ 1 Phase 0 paper) → contribution (one sentence) → falsification prediction → expected outcome.

5. `feasibility_validation` is the canonical feasibility judgment. Its verdicts compare candidate against `intake` user-relative — there is NO absolute compute cap. If the candidate's compute_budget genuinely needs 100 GPU-days for a scaling-law experiment AND `intake.compute` is a large cluster, that's `compute = feasible`. If intake.compute is unspecified, pick a defensible default and note in rationale.

6. **Echo vs reference policy**: the idea-card PDF template renders the following from earlier-phase outputs DIRECTLY (do NOT echo into Phase 4 output): `closest_adjacent` from Phase 1, `collision_report.classified_hits` from Phase 3.1, `lit_grounding_mode` from Phase 0 sentinel. Echo into Phase 4 output ONLY: anti-substitution-guarded fields (falsification + mechanism_probe + theoretical_leg + engineering_leg + differentiation_from_lit) and structurally-rearranged content (reviewer_concerns_and_responses, motivation sections).

7. **No calendar projections anywhere**. NO "by week N", "in the first month", "two-week plan". Sequencing is in dependencies ("step S2 must complete before S3 can run") not weeks.

8. **No experiment matrix, ablation plan, baseline table, expected figures**. The skill produces the IDEA + falsifiability + honest feasibility — experimental engineering is the user's design responsibility.

9. **`domain_landscape` is informational, not generative**. Phase 4 echoes Phase 1's `domain_pattern_distribution` verbatim into `domain_landscape.pattern_distribution` and joins it with the candidate's `pattern_composition` to show saturation per chosen pattern. The `position_note` is one descriptive sentence — NOT prescriptive ("you should/shouldn't use X because saturated"), just descriptive ("primary X is at saturation level Y in this area"). Selection of patterns happened upstream by structural fit; this block just makes the position transparent to the reader.
